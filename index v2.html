L1

XLOOKUP

ARC Group — Penyetaraan Mata Kuliah

›

Buat kolom ARC Group untuk mengetahui padanan dari setiap kode MK, menggunakan data N_COURSE_EQUIVALENCIES.

Formula L1 XLOOKUP
= XLOOKUP( [Kode MK yang dicari], ← nilai yang ingin dicocokkan [Kolom kode di N_COURSE_EQUIV], ← sumber pencarian [Kolom ARC Group di N_COURSE_EQUIV], ← hasil yang ingin ditampilkan "-" ← jika tidak ditemukan, isi ini )

Cara kerja tiap bagian

1 — Dicari

Klik sel kolom Kode MK di data utama. Ini yang dicocokkan.

2 — Sumber

Blok seluruh kolom kode MK di sheet N_COURSE_EQUIVALENCIES.

3 — Hasil

Blok kolom ARC Group di sheet yang sama — ini yang akan muncul jika cocok ditemukan.

4 — Fallback

Tulis "-" agar sel tidak tampil error #N/A. Ini keunggulan XLOOKUP — tidak perlu IFERROR terpisah!

💡
XLOOKUP vs VLOOKUP: Tidak perlu hitung nomor kolom, bisa cari dari kolom mana saja, dan ada fallback bawaan. Jauh lebih simpel!
L2

IFERROR + INDEX + MATCH

Enrollment — Apakah Sudah Daftar Periode Ini?

›

Cek apakah mahasiswa sudah mendaftarkan MK tersebut di periode berjalan. Gunakan data N_PS_STDNT_ENROLL_STATUS_LITE dengan filter Status = E.

Formula L2 IFERROR + INDEX + MATCH
= IFERROR( INDEX( [Kolom hasil di sheet Enrollment], ← kolom yang ingin ditampilkan MATCH( [NIM & Kode MK gabungan], ← nilai yang dicari (2 kriteria!) [Kolom NIM & Kode MK sumber], ← sumber data 0 ← 0 = cari yang PERSIS SAMA ) ), "-" ← tampilkan ini jika tidak ketemu )

Cara kerja 3 fungsi ini

IFERROR

Pelindung error. Kalau formula di dalamnya tidak ketemu, tampilkan "-" bukan pesan error merah.

INDEX

Pengambil nilai. Menerima nomor baris dari MATCH, lalu mengambil nilai dari baris tersebut.

MATCH → 0

Pencari posisi baris. Angka 0 = cari yang persis sama. Hasilnya nomor baris untuk INDEX.

🔗
Untuk cari 2 kriteria, gabungkan NIM + Kode MK dengan & — contoh: A2&B2 untuk nilai dicari, nim_col&mk_col untuk sumber.
⚡
IFERROR bertingkat (2 sumber): Salin blok IFERROR pertama, paste setelah koma pertama menggantikan nilai error, tutup dengan "-".
L3

Opsional — XLOOKUP

Grade Thesis dari 1 Periode Sebelumnya

›

Hanya diperlukan untuk monitoring mahasiswa skripsi. Lookup grade Thesis dari data 1 periode sebelumnya.

Formula L3 XLOOKUP — 1 periode sebelum
= XLOOKUP( [NIM Mahasiswa], [Kolom NIM di data 1 periode sebelum], [Kolom Grade Thesis di data tersebut], "-" )
📌
Data 1 periode sebelum diambil dari N_PS_STDNT_ENROLL_STATUS_LITE, difilter khusus mahasiswa yang memiliki enrollment Thesis. Simpan sebagai sheet terpisah.
L4

IF + OR

Status MK — Sudah Registrasi atau Masih Gagal?

›

Kolom terpenting! Menentukan apakah mahasiswa sudah re-registrasi MK atau masih berstatus Gagal di periode berjalan.

Formula L4 IF + OR
= IF( OR( [Last Term Taken] = [Periode Berjalan], ← kondisi 1 [Kolom Enrollment] <> "-" ← kondisi 2 ), "Sudah Registrasi", ← salah satu kondisi benar "Gagal" ← semua kondisi salah )

🎮 Simulator Logika L4

Pilih skenario untuk melihat hasil kolom Status MK:

Logika kondisi

OR ( )

Benar jika minimal satu kondisi terpenuhi: sudah ambil lagi di periode ini, ATAU sudah ada di data enrollment.

Jika benar

"Sudah Registrasi" — tidak perlu di-follow up.

Jika salah

"Gagal" — masuk daftar yang perlu diingatkan!

L5

XLOOKUP

Status Yudisium Mahasiswa

›

Lookup status kelayakan yudisium dari N_STDNT_GRAD_MONITORING. Penting untuk mendeteksi mahasiswa yang sudah Eligible to Graduate tapi masih punya MK gagal.

Formula L5 XLOOKUP
= XLOOKUP( [NIM Mahasiswa], [Kolom NIM di N_STDNT_GRAD_MONITORING], [Kolom Status Kelayakan Yudisium], "-" )
🎓
Prioritas utama! Eligible to Graduate + MK Gagal = mahasiswa sudah layak wisuda tapi tertahan MK yang belum tuntas. Segera tindak lanjuti!
L6

IFERROR + IF + XLOOKUP + TEXTJOIN

Ketersediaan Kelas & List MK Dibuka

›

Dua kolom sekaligus: (1) cek apakah MK dibuka kelasnya periode ini, (2) gabungkan semua MK yang dibuka dalam satu baris per mahasiswa.

Kolom 1 — cek ketersediaan kelas

Formula ketersediaan IFERROR + IF + XLOOKUP
= IFERROR( IF( XLOOKUP( [Kode MK], [Kolom Kode di N_CLASS_SCHEDULE_MAS], [Kolom Kode di N_CLASS_SCHEDULE_MAS], "-" ) <> "-", ← jika ditemukan di jadwal "Dibuka Kelas", ← tampilkan ini "-" ← jika tidak ada di jadwal ), "-" ← tangkap error apapun )

Kolom 2 — gabungkan list MK yang dibuka

Formula list MK dibuka TEXTJOIN + UNIQUE + FILTER
= TEXTJOIN( "," & CHAR(10), ← pemisah koma + ganti baris 0, ← tampilkan meski ada yang kosong UNIQUE( FILTER( [Kolom Nama/Kode MK], [Kolom Ketersediaan] = "Dibuka Kelas" ) ) )
FILTER

Ambil hanya baris yang Ketersediaan = "Dibuka Kelas".

UNIQUE

Hilangkan duplikat — tiap MK hanya muncul sekali.

TEXTJOIN

Gabungkan jadi satu teks. CHAR(10) = ganti baris agar tampil seperti daftar dalam satu sel.

L7

XLOOKUP × 3

Email, Nomor HP & Status Keaktifan

›

Ambil informasi kontak dari N_MASTER_MAHASISWA. Buat XLOOKUP terpisah untuk setiap kolom yang dibutuhkan.

Formula — ulangi untuk setiap kolom XLOOKUP
= XLOOKUP( [NIM Mahasiswa], [Kolom NIM di N_MASTER_MAHASISWA], [Kolom Email / No. HP / Status], ← ganti sesuai kolom yang dibutuhkan "-" )
📧 Email 📱 Nomor HP ✅ Status Keaktifan

04

Fase Keempat

Output & Tindak Lanjut

Data sudah siap! Pilih jalur output yang sesuai dan tentukan tindak lanjut per mahasiswa.

📊 Jalur A — Pakai Pivot

  1. Buat kolom Status Tindak Lanjut dengan logika IF bertingkat
  2. Blok semua data → Insert Pivot Table
  3. Student ID sebagai nilai dengan COUNT
  4. Status Tindak Lanjut sebagai baris
  5. Klik 2x pada "FU Dibuka Kelas" → buat pesan reminder

🔍 Jalur B — Tanpa Pivot

  1. Filter & hapus Status MK = Enrichment/Thesis/EC/FE
  2. Filter & hapus Status Yudisium = Eligible to Graduate
  3. Filter & hapus Enrollment tidak #N/A
  4. Buat pesan reminder dari sisa data
  5. Bedakan MK Dibuka Kelas vs tidak!

🔀 Logika Status Tindak Lanjut

Klik situasi di bawah untuk melihat status yang tepat:

KondisiStatus Tindak Lanjut
Status Yudisium = Eligible to Graduate ⚡ Sudah Layak Yudisium
Status MK = Gagal DAN Ketersediaan = Dibuka Kelas 📢 FU Dibuka Kelas
Semua kondisi lainnya 📋 FU
