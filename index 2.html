Fase Ketiga

Tambahkan Kolom Lookup

Buat 7 kolom tambahan berikut secara berurutan. Klik setiap kartu untuk melihat formula dan penjelasannya.

L1

XLOOKUP

Kolom ARC Group — Penyetaraan Mata Kuliah

›

Buat kolom baru ARC Group untuk mengetahui padanan (equivalensi) dari setiap kode MK, menggunakan data N_COURSE_EQUIVALENCIES.

Formula XLOOKUP
= XLOOKUP( [Kode MK yang dicari], ← komponen 1: nilai dicari [Kolom kode di N_COURSE_EQUIV], ← komponen 2: sumber pencarian [Kolom ARC Group di N_COURSE_EQUIV], ← komponen 3: hasil yang ditampilkan "-" ← komponen 4: jika tidak ditemukan )

Penjelasan tiap komponen

1 — Nilai Dicari

Klik sel di kolom Kode MK pada data utama. Ini nilai yang akan dicocokkan.

2 — Sumber Data

Blok seluruh kolom kode MK di sheet N_COURSE_EQUIVALENCIES sebagai referensi pencarian.

3 — Hasil

Blok kolom ARC Group di sheet yang sama — ini yang akan muncul jika cocok ditemukan.

4 — Fallback

Tulis "-" agar sel tidak menampilkan error #N/A jika kode MK tidak ada di daftar equivalensi.

💡
Keunggulan XLOOKUP vs VLOOKUP: Tidak perlu menghitung nomor kolom, bisa mencari dari kolom mana saja, dan sudah ada fallback bawaan di komponen ke-4 — tidak perlu IFERROR terpisah.
L2

IFERROR + INDEX + MATCH

Kolom Enrollment — Cek Registrasi Periode Ini

›

Cek apakah mahasiswa sudah meregistrasi MK tersebut di periode berjalan. Lookup ke N_PS_STDNT_ENROLL_STATUS_LITE — ambil hanya baris dengan Status Enrollment = E.

Formula IFERROR + INDEX + MATCH
= IFERROR( INDEX( [Kolom hasil di sheet Enrollment], ← kolom yang ingin ditampilkan MATCH( [NIM & Kode MK gabungan], ← nilai yang dicari [Kolom NIM & Kode MK sumber], ← sumber data pencarian 0 ← 0 = pencarian EXACT/akurat ) ), "-" ← tampilkan ini jika formula ERROR )

Cara kerja tiga fungsi ini

IFERROR

Pelindung error. Jika formula di dalamnya menghasilkan #N/A karena tidak ketemu, tampilkan "-". Bisa ditumpuk 2x untuk 2 sumber berbeda.

INDEX

Pengambil nilai. Blok kolom yang ingin ditampilkan. INDEX menerima nomor baris dari MATCH lalu mengambil nilai di baris tersebut.

MATCH → 0

Pencari posisi baris. Angka 0 = pencarian akurat/exact. Hasilnya berupa nomor baris yang kemudian dipakai oleh INDEX.

🔗
Cari 2 kriteria? Gabungkan NIM + Kode MK dengan & — contoh nilai dicari: A2&B2, sumber: col_nim&col_mk.
⚡
IFERROR bertingkat: Salin blok IFERROR pertama (dari IFERROR sampai sebelum koma terakhir), paste setelah koma pertama menggantikan nilai error, lalu tutup dengan "-".
L3

Opsional — XLOOKUP

Grade Thesis 1 Periode Sebelum

›

Hanya diperlukan untuk monitoring mahasiswa skripsi. Lookup grade Thesis dari data 1 periode sebelumnya.

Formula XLOOKUP — 1 periode sebelum
= XLOOKUP( [NIM Mahasiswa], [Kolom NIM di data 1 periode sebelum], [Kolom Grade Thesis di data tersebut], "-" )
📌
Data 1 periode sebelum diambil dari N_PS_STDNT_ENROLL_STATUS_LITE, difilter khusus mahasiswa yang memiliki enrollment Thesis. Siapkan sebagai sheet terpisah.
L4

IF + OR

Kolom Status Mata Kuliah — Sudah Registrasi / Gagal

›

Kolom paling penting! Menentukan apakah mahasiswa sudah re-registrasi MK tersebut atau masih berstatus gagal di periode berjalan.

Formula IF + OR
= IF( OR( [Last Term Taken] = [Periode Berjalan], ← kondisi 1 [Kolom Enrollment] <> "-" ← kondisi 2 ), "Sudah Registrasi", ← jika salah satu kondisi BENAR "Gagal" ← jika semua kondisi SALAH )

Logika kondisi

OR ( )

BENAR jika minimal satu kondisi terpenuhi: sudah ambil lagi di periode ini, ATAU sudah terdeteksi di data enrollment.

Jika TRUE

Tampilkan "Sudah Registrasi" — mahasiswa sudah mengambil MK kembali, tidak perlu di-follow up.

Jika FALSE

Tampilkan "Gagal" — mahasiswa belum re-registrasi, masuk daftar yang perlu diingatkan.

KondisiHasil Kolom
Last Term Taken = Periode BerjalanSudah Registrasi
Kolom Enrollment ≠ "-"Sudah Registrasi
Mengandung Thesis/Enrichment/EC/FETampilkan kategorinya
Tidak ada kondisi terpenuhiGagal
L5

XLOOKUP

Kolom Status Yudisium

›

Lookup status kelayakan yudisium dari N_STDNT_GRAD_MONITORING. Penting untuk mendeteksi mahasiswa yang sudah Eligible to Graduate tapi masih punya MK gagal — ini prioritas tinggi!

Formula XLOOKUP
= XLOOKUP( [NIM Mahasiswa], [Kolom NIM di N_STDNT_GRAD_MONITORING], [Kolom Status Kelayakan], "-" )
🎓
Eligible to Graduate + MK Gagal = prioritas utama! Mahasiswa ini sudah layak wisuda tapi tertahan oleh MK yang belum tuntas. Pastikan masuk tindak lanjut segera.
L6

IFERROR + IF + XLOOKUP + TEXTJOIN

Ketersediaan Kelas & List MK Dibuka

›

Dua kolom sekaligus: (1) cek apakah MK dibuka kelasnya di periode ini, (2) gabungkan semua MK yang dibuka dalam satu baris per mahasiswa.

Kolom 1 — Cek Ketersediaan IFERROR + IF + XLOOKUP
= IFERROR( IF( XLOOKUP( [Kode MK], [Kolom Kode di N_CLASS_SCHEDULE_MAS], [Kolom Kode di N_CLASS_SCHEDULE_MAS], "-" ) <> "-", ← jika XLOOKUP tidak kembalikan "-" "Dibuka Kelas", ← berarti MK ditemukan di jadwal "-" ← MK tidak ada di jadwal kelas ), "-" ← tangkap error apapun )

Cara kerja formula ketersediaan

XLOOKUP

Cari kode MK di jadwal kelas. Jika ketemu → kembalikan kode MK-nya. Jika tidak → kembalikan "-".

IF

Jika hasil XLOOKUP bukan "-" (artinya ketemu) → tampilkan "Dibuka Kelas". Jika tidak → tampilkan "-".

IFERROR

Tangkap error apapun (misal tipe data tidak cocok) → tampilkan "-" agar sel tidak rusak.

Kolom 2 — Gabungkan List MK Dibuka TEXTJOIN + UNIQUE + FILTER
= TEXTJOIN( "," & CHAR(10), ← pemisah: koma + baris baru 0, ← 0 = tampilkan meski ada yang kosong UNIQUE( FILTER( [Kolom Nama/Kode MK], ← yang ingin ditampilkan [Kolom Ketersediaan] = "Dibuka Kelas" ) ) )

Cara kerja formula list MK

FILTER

Ambil hanya baris dengan Ketersediaan = "Dibuka Kelas", tampilkan kolom Nama MK-nya.

UNIQUE

Hilangkan duplikat dari hasil FILTER — tiap MK hanya muncul sekali.

TEXTJOIN

Gabungkan semua hasil jadi satu teks. CHAR(10) = baris baru, sehingga tampil seperti daftar di satu sel.

L7

XLOOKUP × 3

Email, Telepon & Status Keaktifan

›

Ambil informasi kontak dari N_MASTER_MAHASISWA. Lakukan XLOOKUP terpisah untuk setiap kolom yang dibutuhkan.

Formula — ulangi untuk setiap kolom XLOOKUP
= XLOOKUP( [NIM Mahasiswa], [Kolom NIM di N_MASTER_MAHASISWA], [Kolom Email / No. Telepon / Status Keaktifan], ← ganti tiap kolom "-" )
📧 Email 📱 Nomor Telepon ✅ Status Keaktifan
🎉
Data lengkap! Setelah L7 selesai, semua kolom lookup sudah terisi. Data siap diproses ke tahap output dan tindak lanjut.

Fase Keempat

Output & Tindak Lanjut

📊

Jalur A — Dengan Pivot

  1. Buat kolom Status Tindak Lanjut:
    Kolom Status Yudisium = Eligible to Graduate → Sudah Layak Yudisium Kolom Status Mata Kuliah = Gagal DAN Kolom Ketersediaan = Dibuka Kelas → FU Dibuka Kelas Lainnya → FU
  2. Blok semua data → Insert Pivot Table
  3. Student ID sebagai nilai dengan COUNT
  4. Status Tindak Lanjut sebagai baris
  5. Klik 2x pada FU Dibuka Kelas → buat pesan reminder
🔍

Jalur B — Tanpa Pivot

  1. Filter & hapus Status MK = Enrichment/Thesis/EC/FE
  2. Filter & hapus Status Yudisium = Eligible to Graduate
  3. Filter & hapus Enrollment tidak #N/A
  4. Buat pesan reminder dari sisa data
  5. Bedakan antara MK Dibuka Kelas vs tidak!
🔎
Validasi wajib: Bandingkan jumlah data hasil olahanmu dengan rekan. Kalau sama → selesai dengan benar! Kalau beda → ada langkah yang perlu dicek ulang, sabar ya 🙂
🎉

Proses Selesai!

Data siap digunakan untuk reminder mahasiswa. Selalu bandingkan hasil dengan rekan tim sebelum mengirim pesan.
