03 — Fase Ketiga

Tambahkan Kolom Lookup (L1–L7)

Buat 7 kolom tambahan secara berurutan. Jawab kuis di setiap langkah untuk membuka kolom berikutnya.

0 / 7 kolom selesai
L1
XLOOKUP — ARC Group (Penyetaraan MK)
Dapatkan padanan kode MK dari N_COURSE_EQUIVALENCIES
Aktif
Formula L1 — XLOOKUP
= XLOOKUP( [Kode MK yang dicari], ← nilai yang ingin dicocokkan [Kolom kode di N_COURSE_EQUIV], ← sumber pencarian [Kolom ARC Group di N_COURSE_EQUIV], ← hasil yang ditampilkan "-" ← jika tidak ditemukan )
💡 XLOOKUP vs VLOOKUP: Tidak perlu hitung nomor kolom, bisa cari dari kolom mana saja, dan ada fallback bawaan. Jauh lebih simpel!
Langkah-langkah:
Klik sel kolom Kode MK di data utama sebagai nilai yang dicocokkan
Blok seluruh kolom kode MK di sheet N_COURSE_EQUIVALENCIES
Blok kolom ARC Group di sheet yang sama sebagai hasil
Tulis "-" sebagai fallback agar tidak tampil error #N/A
Kuis L1
Apa keunggulan XLOOKUP dibanding VLOOKUP dalam kasus ini?
L2
IFERROR + INDEX + MATCH — Enrollment Periode Ini
Cek apakah mahasiswa sudah daftar MK periode berjalan
🔒 Terkunci
Formula L2 — IFERROR + INDEX + MATCH
= IFERROR( INDEX( [Kolom hasil di sheet Enrollment], MATCH( [NIM & Kode MK gabungan], ← 2 kriteria digabung! [Kolom NIM & Kode MK sumber], 0 ← cari yang PERSIS SAMA ) ), "-" )
🔗 Untuk cari 2 kriteria, gabungkan NIM + Kode MK dengan & — contoh: A2&B2 untuk nilai dicari, nim_col&mk_col untuk sumber.
Buat formula IFERROR sebagai pelindung error
Gunakan INDEX untuk mengambil nilai dari kolom hasil di sheet Enrollment
Gunakan MATCH dengan gabungan NIM&KodeMK untuk mencari 2 kriteria
Set nilai 0 di MATCH agar pencarian persis sama (exact match)
Kuis L2
Mengapa kita menggabungkan NIM dan Kode MK dalam MATCH?
L3
XLOOKUP Opsional — Grade Thesis 1 Periode Sebelum
Hanya untuk monitoring skripsi
🔒 Terkunci
Formula L3 — XLOOKUP (1 periode sebelum)
= XLOOKUP( [NIM Mahasiswa], [Kolom NIM di data 1 periode sebelum], [Kolom Grade Thesis], "-" )
📌 Data 1 periode sebelum diambil dari N_PS_STDNT_ENROLL_STATUS_LITE, difilter khusus mahasiswa yang memiliki enrollment Thesis. Simpan sebagai sheet terpisah.
Filter data enrollment untuk Thesis periode sebelumnya
Simpan sebagai sheet terpisah
Buat XLOOKUP dengan NIM sebagai kunci pencarian
Kuis L3

L4
IF + OR — Status MK (Sudah Registrasi / Gagal?)
Kolom terpenting — menentukan status mahasiswa
🔒 Terkunci
Formula L4 — IF + OR
= IF( OR( [Last Term Taken] = [Periode Berjalan], ← kondisi 1 [Kolom Enrollment] <> "-" ← kondisi 2 ), "Sudah Registrasi", ← salah satu kondisi benar "Gagal" ← semua kondisi salah )

Kondisi 1: periksa apakah Last Term Taken = Periode Berjalan

L5
XLOOKUP — Status Yudisium Mahasiswa
Deteksi mahasiswa Eligible to Graduate dengan MK gagal
🔒 Terkunci
Formula L5 — XLOOKUP
= XLOOKUP( [NIM Mahasiswa], [Kolom NIM di N_STDNT_GRAD_MONITORING], [Kolom Status Kelayakan Yudisium], "-" )
🎓 Prioritas utama! Eligible to Graduate + MK Gagal = mahasiswa sudah layak wisuda tapi tertahan MK yang belum tuntas. Segera tindak lanjuti!
Gunakan NIM sebagai kunci XLOOKUP ke N_STDNT_GRAD_MONITORING
Ambil kolom Status Kelayakan Yudisium sebagai hasil
Flag mahasiswa dengan status "Eligible to Graduate" sebagai prioritas
Kuis L5
Kenapa mahasiswa "Eligible to Graduate" yang masih punya MK gagal menjadi prioritas?
L6
IFERROR+IF+XLOOKUP + TEXTJOIN — Ketersediaan Kelas
Dua kolom sekaligus: cek kelas dibuka + gabungkan list MK
🔒 Terkunci
Kolom 1 — Cek ketersediaan kelas
= IFERROR( IF( XLOOKUP([Kode MK],[Kolom Kode N_CLASS_SCHEDULE_MAS], [Kolom Kode N_CLASS_SCHEDULE_MAS],"-") <> "-", "Dibuka Kelas", "-" ), "-" )
Kolom 2 — Gabungkan list MK dibuka
= TEXTJOIN(","&CHAR(10), 0, UNIQUE( FILTER([Kolom Nama MK], [Kolom Ketersediaan] = "Dibuka Kelas") ) )
Buat kolom pertama: cek ketersediaan kelas dengan IFERROR+IF+XLOOKUP
Buat kolom kedua: gunakan FILTER untuk ambil MK yang "Dibuka Kelas"
Gunakan UNIQUE untuk hilangkan duplikat dalam list
Gabungkan dengan TEXTJOIN menggunakan CHAR(10) sebagai pemisah baris

04 — Fase Keempat

Output & Tindak Lanjut

Data sudah siap! Pilih jalur output yang sesuai dan ikuti langkah-langkah per alur.

Pilih alur yang akan digunakan:
📊 Alur A — Pakai Pivot
Buat Pivot Table untuk analisis dan summary data
🔍 Alur B — Tanpa Pivot
Filter manual dan buat reminder langsung dari data
📊 Langkah-langkah Alur A — Pakai Pivot
1
Buat kolom Status Tindak Lanjut dengan logika IF bertingkat berdasarkan Status Yudisium dan Status MK
2
Blok semua data → Insert Pivot Table di sheet baru
3
Taruh Student ID sebagai nilai dengan COUNT di area Values
4
Taruh Status Tindak Lanjut sebagai baris (Rows) pada Pivot
5
Klik 2x pada cell "FU Dibuka Kelas" → buat pesan reminder dari data yang muncul
🎉
Proses Alur A Selesai!
Data siap digunakan untuk reminder mahasiswa. Selalu bandingkan hasil dengan rekan tim sebelum mengirim pesan.
🔍 Langkah-langkah Alur B — Tanpa Pivot
1
Filter & hapus baris dengan Status MK = Enrichment / Thesis / EC / FE
2
Filter & hapus baris dengan Status Yudisium = Eligible to Graduate
3
Filter & hapus baris dengan Enrollment tidak #N/A (sudah terdaftar)
4
Buat pesan reminder dari sisa data — bedakan MK Dibuka Kelas vs tidak dibuka
5
Validasi dengan rekan: bandingkan jumlah data hasil. Sama = selesai! Beda = cek ulang langkah yang mungkin terlewat
🎉
Proses Alur B Selesai!
Data siap digunakan untuk reminder mahasiswa. Selalu bandingkan hasil dengan rekan tim sebelum mengirim pesan.
